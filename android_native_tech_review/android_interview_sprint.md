# Android 面试冲刺索引（高频深问 + 追问防守版）

## 使用方式
- 回答顺序：结论 -> 依据 -> 落地 -> 风险。
- 追问应对：先确认场景，再给取舍，不要给绝对答案。

## A. 组件与生命周期（10题）

### 1. Activity 与 Fragment 的职责边界如何划分？
标准答法：Activity 管页面容器、导航和系统回调；Fragment 管局部 UI 与交互；状态放 ViewModel。
追问点：单 Activity 架构下 Activity 还剩哪些职责？
常见陷阱：把网络、缓存、业务决策塞进 Fragment。
案例模板："我把权限与路由集中在 Activity，Fragment 只渲染状态，页面崩溃率和回归成本都下降。"

### 2. Activity 启动模式怎么选？
标准答法：默认 `standard`，通知页常用 `singleTop`，入口汇聚页才考虑 `singleTask`。
追问点：消息通知跳详情如何保证返回路径正确？
常见陷阱：用 launchMode 硬修返回栈，导致路径不可预测。
案例模板："先画用户返回路径，再选 launchMode/flag，避免线上返回栈错乱。"

### 3. Service、WorkManager、协程任务如何取舍？
标准答法：实时持续任务用前台服务；可延迟可靠任务用 WorkManager；短任务用协程。
追问点：同一个同步任务为何不用前台服务？
常见陷阱：所有后台任务都放 Service 保活。
案例模板："把离线上传改 WorkManager 后，系统约束下成功率更稳定，电量投诉下降。"

### 4. BroadcastReceiver 为何不能做重任务？
标准答法：`onReceive` 执行窗口短，重任务会卡主线程引发 ANR。
追问点：那 BOOT_COMPLETED 后复杂初始化怎么办？
常见陷阱：在 `onReceive` 里直接网络请求。
案例模板："Receiver 只做路由，重任务入 WorkManager，并加幂等 key 防重复。"

### 5. ContentProvider 何时必须用？
标准答法：跨进程共享且需权限控制时才用；应用内访问优先 Repository/Room。
追问点：Provider 与 AIDL 的边界？
常见陷阱：应用内也强行走 Provider 增加复杂度。
案例模板："对外共享文件改 FileProvider + 临时授权，规避 file:// 暴露风险。"

### 6. 配置变更重建如何保状态？
标准答法：瞬时状态 Bundle、页面状态 ViewModel、持久状态数据库。
追问点：进程被杀后如何恢复关键状态？
常见陷阱：全部放 ViewModel，进程重启后丢失。
案例模板："用 SavedStateHandle 保存查询条件，旋转和回收后列表位置可恢复。"

### 7. 为何 onCreate 不做重初始化？
标准答法：它在启动关键路径，重活会拖慢首帧和可交互时间。
追问点：哪些初始化必须阻塞？
常见陷阱：三方 SDK 全量同步初始化。
案例模板："拆成阻塞/延迟任务后，冷启动明显下降。"

### 8. 组件导出安全基线？
标准答法：默认不导出；必须导出时加权限与参数白名单。
追问点：仅 exported=false 就够吗？
常见陷阱：导出组件忘记调用方校验。
案例模板："对敏感 Service 增签名权限后，越权调用告警归零。"

### 9. Deep Link 安全风险？
标准答法：参数注入、越权直达、匹配范围过宽。
追问点：已登录态缓存会不会被绕过？
常见陷阱：只校验 host 不校验 path/参数。
案例模板："加 path 白名单+二次鉴权后，外链异常跳转问题消失。"

### 10. 前台服务通知如何合规不扰民？
标准答法：通知可解释、可关闭、类型声明准确。
追问点：Android 13 通知权限拒绝怎么办？
常见陷阱：隐藏通知或误报高优先级。
案例模板："把通知文案改为任务进度+停止入口，投诉明显减少。"

## B. 架构与工程化（10题）

### 1. MVVM 如何防状态爆炸？
标准答法：单一 UiState + 子状态拆分 + 明确事件流。
追问点：复杂页面为何要 reducer？
常见陷阱：View 层拼状态，导致不可追踪。
案例模板："引入状态机后，UI bug 定位时间缩短。"

### 2. Clean Architecture 会过度设计吗？
标准答法：小项目可能过度，中大型项目收益高，关键是按需分层。
追问点：你如何判断要不要引入 UseCase 层？
常见陷阱：机械套模板，层层转发无价值。
案例模板："高复用业务抽 UseCase 后，多端复用和测试都更稳。"

### 3. Repository 的真实职责？
标准答法：数据策略中心，负责本地/远端协调、缓存与错误统一。
追问点：Repository 和 DataSource 如何分界？
常见陷阱：Repository 退化成 Retrofit 包装器。
案例模板："做 SWR 缓存后，首屏体验和接口压力都改善。"

### 4. MVC/MVP 遗留如何迁移？
标准答法：分步迁移：抽数据边界 -> 抽状态边界 -> 抽业务边界。
追问点：迁移期间如何防止双逻辑漂移？
常见陷阱：大爆炸重构导致长周期停滞。
案例模板："按 feature 迁移并双轨对照，业务迭代不停。"

### 5. UI 不应直依 DTO/Entity 的原因？
标准答法：协议/库表变化频繁，直接依赖会放大变更冲击。
追问点：Mapper 会不会增加性能损耗？
常见陷阱：为了省事让 UI 直接拿 Entity。
案例模板："加 UI Model 后后端字段调整不再影响页面层。"

### 6. 可测试架构怎么定义？
标准答法：依赖可替换、业务可脱框架运行、输入输出可断言。
追问点：你如何测 ViewModel 状态流顺序？
常见陷阱：把 Android Context 混进 domain。
案例模板："domain 单测覆盖后，核心规则回归速度大幅提升。"

### 7. 多团队并行如何防架构失控？
标准答法：先定模块边界和契约，再用 lint/评审门禁执行。
追问点：跨模块紧急需求如何快速协同？
常见陷阱：直接跨层调用内部实现。
案例模板："契约化接口后，团队并行冲突显著减少。"

### 8. 何时选 MVI 而非 MVVM？
标准答法：状态复杂、回放要求高选 MVI；常规页面 MVVM 足够。
追问点：MVI 的主要成本是什么？
常见陷阱：所有页面强行上 MVI。
案例模板："交易页改 MVI 后状态一致性问题明显收敛。"

### 9. 技术债治理机制怎么建？
标准答法：分级+预算+指标跟踪，纳入迭代制度化执行。
追问点：业务压力大时如何保证偿还？
常见陷阱：只建清单不追结果。
案例模板："把高危债务做发布门禁后，线上事故减少。"

### 10. 架构优化如何量化收益？
标准答法：用交付效率、线上质量、稳定性三类指标做前后对比。
追问点：如何排除业务复杂度波动影响？
常见陷阱：只讲主观感受不讲数据。
案例模板："优化后需求周期和回归缺陷率都有统计下降。"

## C. 并发与稳定性（10题）

### 1. 协程如何避免泄漏和悬挂任务？
标准答法：结构化并发 + 生命周期作用域 + 可见性收集。
追问点：`SupervisorJob` 在哪类场景更合适？
常见陷阱：GlobalScope 启动长期任务。
案例模板："统一 scope 后页面退出残留任务几乎消失。"

### 2. Flow 与 LiveData 如何分工？
标准答法：新功能用 StateFlow/SharedFlow，旧模块渐进兼容 LiveData。
追问点：一次性事件如何防重复消费？
常见陷阱：用 LiveData 传事件导致粘性问题。
案例模板："事件改 SharedFlow 后重复弹窗问题消失。"

### 3. ANR 排查顺序？
标准答法：先 trace 定位，再归因分类，再验证修复。
追问点：无 trace 时怎么补证据？
常见陷阱：凭经验盲改线程。
案例模板："定位到主线程锁等待后改锁粒度，ANR 下降。"

### 4. 并发竞态导致状态错乱怎么处理？
标准答法：单写入口+版本戳/请求 ID 去重+互斥控制。
追问点：多请求并发返回顺序乱了怎么办？
常见陷阱：最后返回覆盖最新状态。
案例模板："加 requestId 后旧响应不再覆盖新数据。"

### 5. 线程池参数如何设计？
标准答法：按任务类型拆池，并限制队列和拒绝策略。
追问点：CPU 池大小为何不盲目等于核心数？
常见陷阱：一个线程池承载全部任务。
案例模板："下载池独立后，主业务请求延迟更稳定。"

### 6. 为何不能吞掉 CancellationException？
标准答法：会破坏取消语义，导致状态与资源清理异常。
追问点：finally 里还能做哪些动作？
常见陷阱：catch(Exception) 后直接忽略取消。
案例模板："修复后任务取消链路恢复正常。"

### 7. 高频输入如何做背压？
标准答法：`debounce + distinctUntilChanged + flatMapLatest`。
追问点：实时搜索和联想输入参数如何调？
常见陷阱：每次输入都发请求。
案例模板："搜索请求数下降后，页面流畅度提升。"

### 8. 弱网重试怎么防雪崩？
标准答法：幂等请求指数退避+抖动，非幂等谨慎重试。
追问点：重试上限怎么设？
常见陷阱：所有错误都无限重试。
案例模板："引入退避后网关故障期流量峰值受控。"

### 9. 线上稳定性守护怎么做？
标准答法：指标基线+发布门禁+分级告警+灰度放量。
追问点：门禁阈值如何确定？
常见陷阱：上线后只看 crash 不看 ANR。
案例模板："灰度观察机制让高风险版本可提前回滚。"

### 10. 偶发线上问题如何复现？
标准答法：先补埋点证据，再做故障注入和回放。
追问点：低概率问题如何提升复现概率？
常见陷阱：只在本机常规网络测试。
案例模板："弱网+低内存组合场景复现并修复卡死。"

## D. 数据与存储（10题）

### 1. Room 迁移如何安全可验证？
标准答法：显式 migration + 迁移测试 + 回滚预案。
追问点：大版本迁移如何做灰度？
常见陷阱：线上启 destructive migration。
案例模板："迁移测试覆盖旧版本样本后再全量发布。"

### 2. Room 与 SQLite 何时各自更优？
标准答法：常规业务选 Room，极限性能或重遗留场景选 SQLite。
追问点：SQLite 场景下如何保证维护性？
常见陷阱：仅因“熟悉 SQL”放弃 Room 校验能力。
案例模板："核心模块保留 SQLite，外围模块逐步 Room 化。"

### 3. 事务边界如何定义？
标准答法：以领域一致性为边界，一次业务成功必须原子提交。
追问点：跨库操作如何补偿？
常见陷阱：把一条业务链拆成多个无事务写入。
案例模板："订单主从表入库统一事务后脏数据下降。"

### 4. 索引设计如何平衡读写？
标准答法：以高频查询为导向，配合 explain 验证。
追问点：复合索引顺序如何定？
常见陷阱：为每列都建索引。
案例模板："优化复合索引后查询延迟降低。"

### 5. 本地缓存策略怎么选？
标准答法：强一致直连、可陈旧用 SWR、离线优先本地。
追问点：缓存失效如何精确控制？
常见陷阱：全局统一 TTL 导致数据陈旧或抖动。
案例模板："首页改 SWR 后首屏更快且数据可刷新。"

### 6. 如何避免 N+1 查询？
标准答法：批量查询、预加载关系、合并 SQL。
追问点：何时宁可多次查询也不 join？
常见陷阱：循环里每次查询单条数据。
案例模板："聚合查询替换循环查询后耗时大幅下降。"

### 7. 数据模型演进如何防兼容事故？
标准答法：向前兼容字段+默认值+灰度双读写。
追问点：历史脏数据如何清理？
常见陷阱：直接删旧字段导致旧数据崩溃。
案例模板："双写观察期结束再切主模型，迁移更稳。"

### 8. 为什么不能线上 destructive migration？
标准答法：会清空用户数据，风险不可接受。
追问点：测试环境可以用吗？
常见陷阱：把测试策略带到生产。
案例模板："曾经误用导致数据丢失，后来改显式迁移门禁。"

### 9. 本地敏感数据如何保护？
标准答法：分级存储，高敏数据加密+最小驻留。
追问点：token 是否应长期落盘？
常见陷阱：明文写 SharedPreferences。
案例模板："改用加密存储后安全扫描高危项清零。"

### 10. 离线优先如何设计？
标准答法：读本地、写排队、后台同步、冲突合并。
追问点：冲突策略怎么定业务可接受？
常见陷阱：离线写入不记录操作日志。
案例模板："加操作日志后断网恢复的数据一致性更好。"

## E. 网络与安全（10题）

### 1. 统一错误模型有什么价值？
标准答法：统一 UI 处理、重试策略和监控口径。
追问点：业务错误码如何映射？
常见陷阱：每层定义一套错误结构。
案例模板："统一后告警分类和定位效率明显提升。"

### 2. Token 刷新如何防并发 401 风暴？
标准答法：单飞刷新，其他请求排队等待。
追问点：刷新失败如何收敛用户状态？
常见陷阱：每个请求独立刷新，触发雪崩。
案例模板："接入单飞后 401 峰值期请求量受控。"

### 3. Pinning 失效如何平滑处理？
标准答法：双 pin 轮转 + 灰度 + 紧急开关。
追问点：紧急开关如何防被滥用？
常见陷阱：只配一个 pin。
案例模板："证书轮换期通过双 pin 平稳过渡。"

### 4. 请求可观测最少字段？
标准答法：traceId、API、耗时、网络类型、HTTP/业务码、失败原因。
追问点：如何把客户端和服务端 trace 打通？
常见陷阱：只有日志无统一 requestId。
案例模板："打通 trace 后慢接口定位效率显著提升。"

### 5. 权限申请最佳体验？
标准答法：按需申请、场景化说明、拒绝可降级。
追问点：永久拒绝后如何引导？
常见陷阱：首次启动全量弹权限。
案例模板："改按需申请后授权转化率提升。"

### 6. 导出组件越权怎么防？
标准答法：默认不导出，导出必鉴权+参数白名单。
追问点：第三方集成必须导出怎么办？
常见陷阱：只靠 exported 配置，忽略运行时校验。
案例模板："加签名权限后恶意调用被阻断。"

### 7. WebView 主要风险？
标准答法：JS bridge、不可信 URL、混合内容、文件访问。
追问点：H5 必须调用原生能力如何安全做？
常见陷阱：全局开启 JS 和文件访问。
案例模板："桥接接口改白名单域名后风险收敛。"

### 8. 为什么混淆不等于安全？
标准答法：混淆仅增加逆向成本，不能替代鉴权和数据保护。
追问点：那客户端安全的最小闭环是什么？
常见陷阱：把安全完全寄托在混淆。
案例模板："引入服务端签名校验后伪造请求显著下降。"

### 9. 供应链风险如何治理？
标准答法：锁版本、漏洞扫描、SBOM 追踪、发布门禁。
追问点：紧急高危漏洞如何处置？
常见陷阱：使用动态版本如 `1.2.+`。
案例模板："SCA 门禁拦截过高危依赖进入 release。"

### 10. 安全发布门禁怎么定义？
标准答法：高危项阻断发布，低危项告警并限期偿还。
追问点：门禁误报如何处理？
常见陷阱：安全检查仅做报告不阻断。
案例模板："将明文密钥和导出组件问题设为硬阻断后，发布质量更稳。"

## 收尾高分模板
- 结论：我会优先选什么方案。
- 依据：从稳定性、成本、演进性给三点理由。
- 落地：给一个项目动作和结果。
- 边界：说明不适用场景及备选方案。
