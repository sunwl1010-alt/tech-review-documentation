# Android 面试冲刺索引（高频深问 + 答题框架）

## 使用方式
- 每道题按“结论 -> 原因 -> 取舍 -> 落地案例”作答。
- 回答控制在 60~90 秒，先给结论再展开。

## A. 组件与生命周期（10题）
1. Activity 与 Fragment 的职责边界如何划分？
答题框架：页面容器职责、状态持有、导航边界、反例。
2. Activity 启动模式在真实项目中怎么选？
答题框架：返回栈体验、通知跳转、deep link 汇聚。
3. Service、WorkManager、协程任务如何取舍？
答题框架：实时感知、可靠执行、系统约束。
4. BroadcastReceiver 为什么不能做重任务？
答题框架：执行时机、ANR 风险、任务转交。
5. ContentProvider 何时必须用？
答题框架：跨进程共享、权限模型、替代方案。
6. 配置变更导致重建如何保证状态连续？
答题框架：SavedState、ViewModel、持久层分工。
7. 为什么 onCreate 里不该做重初始化？
答题框架：启动路径、主线程预算、延迟加载。
8. 组件导出安全怎么做基线？
答题框架：默认不导出、权限保护、调用方校验。
9. Deep Link 常见安全风险有哪些？
答题框架：参数校验、越权访问、登录态校验。
10. 前台服务通知如何做到“合规 + 不打扰”？
答题框架：类型声明、可解释文案、可控生命周期。

## B. 架构与工程化（10题）
1. MVVM 在复杂页面下如何防止状态爆炸？
答题框架：状态分层、子状态拆分、事件归一化。
2. Clean Architecture 是否会过度设计？
答题框架：团队规模、变更频率、模块复杂度。
3. Repository 的真实职责是什么？
答题框架：数据聚合、缓存策略、错误归一化。
4. MVC/MVP 遗留项目如何渐进迁移？
答题框架：提边界、双轨运行、分模块下线。
5. 为什么 UI 层不应直接依赖 DTO/Entity？
答题框架：模型污染、演进成本、Mapper 价值。
6. 如何定义“可测试架构”？
答题框架：依赖反转、纯函数边界、可替换实现。
7. 多团队并行开发如何避免架构失控？
答题框架：模块边界、API 契约、评审门禁。
8. 什么时候引入 MVI 而不是 MVVM？
答题框架：状态复杂度、回放需求、学习成本。
9. 如何建立技术债治理机制？
答题框架：债务分级、偿还窗口、收益追踪。
10. 架构优化如何量化收益？
答题框架：交付效率、缺陷率、回归成本指标。

## C. 并发与稳定性（10题）
1. 协程如何避免内存泄漏与悬挂任务？
答题框架：结构化并发、生命周期收集、取消传播。
2. Flow 与 LiveData 在项目中的分工？
答题框架：状态流、一次性事件、兼容迁移。
3. ANR 的系统排查顺序是什么？
答题框架：trace 定位、类型归因、修复闭环。
4. 如何处理并发竞态导致的状态错乱？
答题框架：单写原则、互斥、状态机约束。
5. 线程池参数应该如何设计？
答题框架：任务类型分池、队列策略、拒绝策略。
6. 为什么不能吞掉 CancellationException？
答题框架：协程取消语义、资源泄漏风险。
7. 高频输入场景怎么做背压控制？
答题框架：debounce、distinctUntilChanged、flatMapLatest。
8. 弱网重试怎么避免雪崩？
答题框架：指数退避、抖动、幂等控制。
9. 如何做线上稳定性守护？
答题框架：崩溃/ANR 指标、发布门禁、告警机制。
10. 偶发线上问题如何复现？
答题框架：埋点补证据、故障注入、灰度回放。

## D. 数据与存储（10题）
1. Room 迁移如何做到安全可验证？
答题框架：显式 migration、迁移测试、回滚预案。
2. Room 与直接 SQLite 何时各自更优？
答题框架：维护成本、性能诉求、遗留约束。
3. 事务边界如何定义？
答题框架：领域一致性、原子操作、失败补偿。
4. 索引设计怎么平衡读写性能？
答题框架：查询频率、写放大、explain 验证。
5. 本地缓存策略怎么选？
答题框架：强一致、最终一致、过期模型。
6. 如何避免 N+1 查询问题？
答题框架：批量查询、预加载、SQL 重构。
7. 数据模型演进如何避免兼容事故？
答题框架：向前兼容字段、默认值策略、灰度发布。
8. 为什么不能线上用 destructive migration？
答题框架：数据损失风险、恢复成本、合规风险。
9. 本地存储中的敏感数据如何保护？
答题框架：分级存储、密钥托管、最小驻留。
10. 离线优先策略如何设计？
答题框架：本地优先读取、后台同步、冲突合并。

## E. 网络与安全（10题）
1. 网络层统一错误模型有什么价值？
答题框架：UI 简化、可观测一致、重试策略统一。
2. Token 刷新如何避免并发 401 风暴？
答题框架：单飞刷新、请求排队、失败快速返回。
3. 证书 Pinning 失效如何平滑处理？
答题框架：双 pin 轮转、灰度切换、紧急降级。
4. 网络请求可观测性最少需要哪些字段？
答题框架：traceId、耗时、网络类型、状态码。
5. Android 权限申请的最佳用户体验是什么？
答题框架：按需申请、拒绝降级、二次引导。
6. 组件导出导致越权怎么防？
答题框架：不导出、权限校验、参数白名单。
7. WebView 主要安全风险有哪些？
答题框架：JS bridge、来源校验、混合内容策略。
8. 为什么混淆不能等同于安全？
答题框架：威胁模型、服务端校验、流程防护。
9. 依赖供应链风险如何治理？
答题框架：锁版本、漏洞扫描、SBOM。
10. 如何定义安全发布门禁？
答题框架：高危项阻断、自动扫描、人工复核。

## 收尾模板（通用）
- 结论：先给明确取舍结论。
- 依据：给 2~3 个关键技术依据。
- 落地：给项目实践与指标收益。
- 风险：说明边界与替代方案。
